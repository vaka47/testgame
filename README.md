# Крестики-нолики с промокодом и Telegram-ботом

Небольшая web-игра, где пользователь играет в крестики-нолики против компьютера.  
При победе генерируется 5-значный промокод, показывается на экране и отправляется сообщение в Telegram-бот.  
При проигрыше отправляется сообщение «Проигрыш» и показывается предложение сыграть ещё раз.

## Локальный запуск

1. Установите зависимости:

```bash
npm install
```

2. Создайте файл `.env` в корне проекта:

```env
TELEGRAM_BOT_TOKEN=ваш_токен_бота
TELEGRAM_CHAT_ID=ваш_chat_id
PORT=3000
```

3. Запустите сервер:

```bash
npm start
```

4. Откройте в браузере:

- `http://localhost:3000`

## Telegram-бот

- Бота можно создать через `@BotFather` и получить токен `TELEGRAM_BOT_TOKEN`.
- `TELEGRAM_CHAT_ID` можно взять из любого апдейта, отправленного боту, либо через сторонние инструменты.

Сообщения, которые отправляются:

- При победе игрока: `Победа! Промокод выдан: XXXXX`
- При проигрыше: `Проигрыш`

## Деплой на Vercel (рекомендуемый вариант)

Проект адаптирован под Vercel:

- Статические файлы лежат в `public/` и отдаются как фронтенд.
- API-запросы с фронта идут на `/api/telegram/event` — это serverless‑функция в `api/telegram/event.js`.

Шаги:

1. Залейте проект на GitHub.
2. Зайдите на https://vercel.com → *Add New Project* → выберите репозиторий.
3. В настройках проекта:
   - Framework Preset: `Other`.
4. В разделе *Environment Variables* добавьте:
   - `TELEGRAM_BOT_TOKEN=...`
   - `TELEGRAM_CHAT_ID=...`
5. Нажмите *Deploy*.

Через пару минут вы получите ссылку вида:

- `https://<имя-проекта>.vercel.app`

Эта ссылка — рабочая ссылка на игру для тестового задания.

## Как выложить проект на GitHub

В корне проекта:

```bash
git init
git add .
git commit -m "Initial game implementation"
git branch -M main
git remote add origin https://github.com/<your-username>/<repo-name>.git
git push -u origin main
```

После этого можно подключать репозиторий к Vercel, как описано выше.
